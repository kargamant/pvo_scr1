# Полный путь тестов  

# Память  
- DDR2 на 128Мб для данных и инструкций  
- Двухпортовая TCM на 64Кб для данных и инструкций(**не использует smartconnect**)    
- Блочная память BRAM на 64Кб для данных и инструкций(0xFFFF0000–0xFFFFFFFF)

# Загрузчик попадает на плату   

После генерации битстрима, с помощью mem_update.tcl происходят следующие вещи:  
1. Компиляция кода загрузчика  
2. Из результата сборки формируется файл scbl.mem с текстовым представлением содержимого памяти  
3. Загрузка scbl.mem в BRAM с адреса 0xFFFF0000(сначала идёт секция startup - настройка CSR, SP и т.д., потом vectors с самими инструкциями загрузчика)  
   Глобально заполняется секция инициализации блочной памяти битами из scbl.mem файла

# Работа загрузчика  

1. После CPU_RESET, PC загружается значением 0xFFFFFF00 - SCR1_ARCH_RST_VECTOR в файле scr1_arch_custom.svh  
   По этому адресу располагается секция vectors с инструкциями загрузчика. В этом загрузчике процессор и крутится(можно сказать, что это как очень упрощённая ОС, но скорее BIOS)
```
always_ff @(negedge rst_n, posedge clk) begin
    if (~rst_n) begin
        pc_curr_ff <= SCR1_RST_VECTOR;
    end else if (pc_curr_upd) begin
        pc_curr_ff <= pc_curr_next;
    end
end
```

2. Загрузчик инициализирует uart, таймер и семисегментные светодиоды  

3. Далее по uart выводит информацию о вендоре и ожидает по uart ввод

4. При вводе опции 1, принимает по uart адрес и ожидает передачи по xmodem(для этого там написана отдельная функция по обработке xmodem пакетов)
    Пакет xmodem:
    SOH/STX | 128/1024 байта данных | CRC16 
    На каждый пакет ПЛИС отправляет acknowledge
    В конце передачи хостом отправляется EOT

5. При вводе опции g, принимает по uart адрес и делает jalr на него
```
    uart_putc('\n');
    sc1f_uart_tx_flush();
    rtc_delay_us(20000);
    asm volatile ("jalr %0" :: "r"(arg));
```

Все тесты записываются в TCM самим бутлоадером.
TCM - реализована как блочная память.

Супер коротко:
Сначала прошивка модифицируется tcl-скриптом - изменяется секция инициализации блочной памяти на содержимое scbl.mem
Потом модифицированная прошивка загружается на ПЛИС
После сброса, в PC грузится адрес 0xFFFFFF00, по нему как раз лежит бутлоадер

Всё, дальше проц работает уже в рамках бутлоадера, который коммуницирует с хостом через uart и xmodem
Пишет тесты в TCM и исполняет их путём простого джампа
