# Маршрут загрузки и исполнения бинарника (Nexys4DDR, UART)

> Источники: `fpga/nexys4ddr/scr1/README.md`, `fpga/nexys4ddr/scr1/src/nexys4ddr_scr1.sv` (SoC top), `fpga/nexys4ddr/scr1/src/scr1_arch_custom.svh` (reset vector, карта), `fpga/nexys4ddr/scr1/{scbl.mem, mem_update.tcl}` (образ загрузчика), прошивка загрузчика `sw/sc-bl/**` (scbl.c, xmodem.c, uart.c, scbl.ld), платформа `sw/sc-bl/src/plf_nexys4ddr_scr1.h`.

Этот документ отвечает на вопрос: **как программа физически попадает в SCR1 и начинает исполняться**. Коротко: на плате «зашит» маленький **загрузчик** (SCR Loader) в встроенной SRAM; после сброса ядро стартует с него; загрузчик по UART принимает ваш бинарник по протоколу **XMODEM**, кладёт его в память (DDR или TCM) и передаёт на него управление командой `g` (инструкция `jalr`).

---

## 0. Маршрут в одном взгляде

```
   ┌─────────┐  USB   ┌──────────┐  UART(TTL)  ┌──────────────── FPGA (Artix-7) ────────────────┐
   │   ПК    │◄──────►│  FTDI    │◄───────────►│  axi_uart16550 @0xFF010000                      │
   │ терминал│ 115200 │ FT2232   │ FTDI_TXD/RXD│        │ AXI (cpu_clk = 30 МГц)                  │
   │ +XMODEM │  8N1   └──────────┘             │        ▼                                        │
   └─────────┘                                 │   SCR1 core ── исполняет ЗАГРУЗЧИК из OCRAM      │
                                               │        │        (reset vector 0xFFFFFF00)       │
                                               │        ▼ пишет принятые байты                    │
                                               │   DDR2 @0x0 (128 МБ)  или  TCM @0xF0000000 (64К) │
                                               └─────────────────────────────────────────────────┘
```

1. **Сброс** → SCR1 стартует с адреса `0xFFFFFF00` → это встроенная SRAM (OCRAM) с прошитым загрузчиком.
2. Загрузчик инициализирует **UART (115200 8N1)** и выводит меню.
3. Команда `1` → загрузчик принимает бинарник по **XMODEM** и пишет его в память по указанному адресу.
4. Команда `g` → загрузчик делает `jalr` на этот адрес — ваша программа начинает исполняться.

## 1. Аппаратная основа и тактовые частоты

Плата **Digilent Nexys4DDR** (Xilinx Artix-7). Из `nexys4ddr_scr1.sv` (top-level) видны все внешние сигналы:

```systemverilog
module nexys4ddr_scr1 (
    input  logic CLK100MHZ,      // тактовый генератор платы — 100 МГц
    input  logic CPU_RESETn,     // кнопка сброса
    // ... DDR2 ...
    input  logic FTDI_TXD,       // от ПК (через встроенный USB-UART мост FTDI)
    output logic FTDI_RXD        // к ПК
);
```

Внутри инстанцируются два блока: ядро `scr1_top_axi` (собственно SCR1 с AXI-интерфейсами) и `nexys4ddr_sopc` — платформенный SoC (Vivado block design), где живут генератор тактов, контроллер DDR2 (MIG), UART, встроенная SRAM, таймер и регистры-идентификаторы.

**Частоты — их четыре, не путать:**

| Частота | Значение | Что это |
|---|---|---|
| Осциллятор платы | **100 МГц** | вход `CLK100MHZ` |
| `cpu_clk` (ядро + AXI + UART) | **30 МГц** | генерится MMCM внутри SOPC из 100 МГц; `SCR1_PTFM_CORE_CLK_FREQ = 30000000` |
| Скорость UART (baud) | **115200 бит/с** | скорость поступления данных по линии |
| RTC timebase | 1 МГц | системный таймер |

UART-мост: на плате стоит микросхема **FTDI FT2232**, дающая из USB виртуальный COM-порт. В FPGA её линии приходят как `FTDI_TXD`/`FTDI_RXD` и просто прокидываются на UART SoC:

```systemverilog
assign uart_rxd = FTDI_TXD;   // ПК → SoC
assign FTDI_RXD = uart_txd;   // SoC → ПК
```

## 2. Карта памяти

Из `README.md` проекта (авторитетно для этой сборки; `plf_nexys4ddr_scr1.h` — её же вид со стороны ПО):

| Базовый адрес | Размер | Имя | Описание |
|---|---|---|---|
| `0x00000000` | 128 МБ | **SDRAM** | Внешняя DDR2 (главная память для программ) |
| `0xF0000000` | 64 КБ | **TCM** | Tightly-Coupled Memory ядра (быстрая, детерминированная) |
| `0xF0040000` | 32 Б | Timer | Регистры таймера SCR1 |
| `0xFF000000` | 4 КБ | SOC_ID | 32-битный ID SoC |
| `0xFF001000` | 4 КБ | BLD_ID | 32-битный build ID |
| `0xFF002000` | 4 КБ | CORE_CLK_FREQ | частота ядра (её читает ПО: `PLF_CPU_FREQ`) |
| `0xFF010000` | 4 КБ | **UART** | 16550 UART (IRQ линия 0) |
| `0xFFFF0000` | 64 КБ | **SRAM (OCRAM)** | Встроенная SRAM с прошитым **загрузчиком**; сюда же указывают reset vector и MTVEC |

Адреса TCM/таймера задаются в `scr1_arch_custom.svh` через маску+паттерн (маска `0xFFFF0000` → регион 64 КБ):

```systemverilog
parameter bit [31:0] SCR1_ARCH_RST_VECTOR   = 'hFFFFFF00;  // адрес старта после сброса
parameter bit [31:0] SCR1_ARCH_MTVEC_BASE   = 'hFFFFFF80;  // база вектора прерываний/трапов
parameter bit [31:0] SCR1_TCM_ADDR_MASK     = 'hFFFF0000;  // TCM: 64 КБ
parameter bit [31:0] SCR1_TCM_ADDR_PATTERN  = 'hF0000000;
```

> Обрати внимание: reset vector здесь **`0xFFFFFF00`**, а не `0x200` (как в дефолтном `scr1_arch_description.svh`). Для FPGA-сборки действует именно этот кастомный файл — старт из OCRAM с загрузчиком.

## 3. Загрузчик (SCR Loader): где живёт и как попал в чип

Загрузчик — это отдельная прошивка `sw/sc-bl/` («SCR* first stage boot loader»). Его линкер-скрипт (`scbl.ld`) размещает всё в 64-КБ окне на самом верху адресного пространства — ровно в OCRAM:

```
MEMORY { RAM (rwx) : ORIGIN = 0-64K, LENGTH = 64K }   // 0-64K == 0xFFFF0000, длина 0x10000
...
.startup ORIGIN(RAM)+LENGTH(RAM)-0x800 : { KEEP(*(.startup)); }   // 0xFFFFF800
.vectors ORIGIN(RAM)+LENGTH(RAM)-0x100 : { KEEP(*(.vectors)); }   // 0xFFFFFF00 ← reset vector!
```

То есть `.vectors` попадает точно на `0xFFFFFF00`, а `.startup` — на `0xFFFFF800`. Всё сходится с reset vector.

**Как загрузчик оказывается в SRAM.** Он скомпилирован в файл `scbl.mem` (обычный текстовый hex, `@адрес` + слова), который «вшивается» в содержимое block RAM **после** генерации битстрима — из-за особенности Vivado (Xilinx AR #63042 — начальное содержимое памяти нельзя задать до синтеза). Это делает `mem_update.tcl`:

```tcl
set mem_file  "../scbl.mem"
set sram_cell "blk_mem_gen_0"                 # ← та самая встроенная SRAM (OCRAM)
...
write_mmi $sram_cell $mmi_file
exec updatemem -force --meminfo $mmi_file --data $mem_file --bit $bit_file \
     --proc "dummy" --out $out_file           # ← вписывает scbl.mem в битстрим
```

На выходе — `nexys4ddr_scr1_top_new.bit` (и `.mcs` для FLASH), в котором OCRAM уже инициализирована загрузчиком. Именно этот `_new.bit` и прошивают в плату.

## 4. Сброс → старт загрузчика

Последовательность после подачи питания / нажатия сброса:

1. SoC формирует сбросы (`pwrup_rst_n`, `hard_rst_n` — с фильтром дребезга на 16 тактов в `nexys4ddr_scr1.sv`).
2. SCR1 выходит из сброса и **выбирает первую инструкцию с reset vector `0xFFFFFF00`**. По карте это OCRAM → там лежит `.vectors`/`.startup` загрузчика.
3. Стартовый код (`startup.S`) настраивает стек и прыгает в `main()` загрузчика (`scbl.c`):

```c
int main(void) {
    scr_rtc_init();
    sc1f_uart_init();     // ← инициализация UART (115200 8N1)
    sc1f_leds_init();
    show_header();        // печатает "SCR loader v1.2-..."
    hwinfo();             // ISA, частоты, карта памяти — всё по UART
    usage();              // меню команд
    while (1) { ... }     // ожидание команд по UART
}
```

С этого момента загрузчик «разговаривает» с вами через терминал на ПК.

## 5. UART: частота, формат, регистры

UART — 16550-совместимый (`PLF_UART0_16550`), по адресу `0xFF010000` (`PLF_MMIO_BASE + 0x10000`). Инициализация (`uart.c`) — классическая для 16550:

```c
void sc1f_uart_init(void) {
    sc1f_uart_write(..., SC1F_UART_IER, 0);                 // выкл. прерывания
    sc1f_uart_write(..., SC1F_UART_MCR, 0);
    // задать baud: открыть divisor latch (DLAB) и записать делитель
    sc1f_uart_write(..., SC1F_UART_CONTROL, SC1F_UART_LCR_INIT | SC1F_UART_LCR_DIVL);
    sc1f_uart_write(..., SC1F_UART_DIV_LO,  UART_115200_CLK_DIVISOR & 0xff);
    sc1f_uart_write(..., SC1F_UART_DIV_HI, (UART_115200_CLK_DIVISOR >> 8) & 0xff);
    sc1f_uart_write(..., SC1F_UART_CONTROL, SC1F_UART_LCR_INIT);  // 8N1, DLAB off
    // включить и сбросить FIFO
    sc1f_uart_write(..., SC1F_UART_FCR, SC1F_UART_FCR_R_RST | SC1F_UART_FCR_T_RST);
    sc1f_uart_write(..., SC1F_UART_FCR, SC1F_UART_FCR_INIT);
}
```

**Формат кадра:** `SC1F_UART_LCR_INIT = SC1F_UART_LCR_CL8` → **8 бит данных, без чётности, 1 стоп-бит = 8N1**.

**Скорость:** `PLF_UART_BAUDRATE = 115200`. Делитель для 16550 (частота опорного клока / (16 × baud)):

```c
#define UART_CLK_FREQ  PLF_CPU_FREQ              // = 30 МГц (частота AXI/ядра)
#define UART_115200_CLK_DIVISOR  (UART_CLK_FREQ / UART_BAUD_RATE + 7) / 16
```

Подставляем: `(30000000 / 115200 + 7) / 16 = (260 + 7)/16 = 16`. При делителе 16 фактическая скорость `30e6/(16×16) = 117187` бод — отклонение +1.7 % от 115200, в пределах допуска UART. Опорный клок UART — те самые **30 МГц** (UART сидит на AXI-шине с `cpu_clk`).

**Пропускная способность:** 115200 бод при 8N1 (10 бит на байт) ≈ **11.5 КБ/с** «сырых». С накладными XMODEM (заголовок + CRC + ACK на каждый пакет) реально ~10–11 КБ/с. То есть 64-КБ программа грузится ~6 секунд, а мегабайтная — около двух минут.

Регистры доступа (из `uart.h`): статус `0xFF010000+0x14` (бит 0 — «есть принятый байт», бит 5 — «можно передавать»), данные RX/TX по смещению `0x00`.

## 6. Приём бинарника по UART — протокол XMODEM

Команда меню `1` (`cmd_xload`) вызывает `xmodem_receive(dest, MAX_XMODEM_RX_LEN)`, где `dest` — **адрес, который вы ввели** (куда класть программу), а лимит = `PLF_MEM_SIZE` (128 МБ DDR). Это классический **XMODEM-CRC** с поддержкой 1К-пакетов (XMODEM-1K).

### Как это работает (из `xmodem.c`)

**1. Инициация — приёмник просит отправителя начать.** Приёмник (плата) шлёт символ `'C'` (0x43) раз в ~1.5 с, сигнализируя «я готов, режим с CRC»:

```c
unsigned char trychar = 'C';
...
if (trychar) _outbyte(trychar);     // периодически шлём 'C'
c = _inbyte((DLY_1S)*3/2);          // ждём ответ
```

**2. Приём пакетов.** Отправитель (терминал на ПК) шлёт пакеты. Первый байт задаёт размер:

```c
case SOH:  pktsize = 128;  goto start_recv;    // 0x01 → пакет 128 байт
case STX:  pktsize = 1024; goto start_recv;    // 0x02 → пакет 1024 байта (XMODEM-1K)
case EOT:  _outbyte(ACK); return totalbytes;   // 0x04 → конец передачи
```

Структура одного пакета:

```
┌──────┬───────┬────────┬───────────── data ─────────────┬───────┬───────┐
│ SOH/ │ seq#  │ ~seq#  │  128 или 1024 байта полезных    │ CRC16 │ CRC16 │
│ STX  │       │        │  данных программы               │  hi   │  lo   │
└──────┴───────┴────────┴────────────────────────────────┴───────┴───────┘
```

Приёмник построчно (побайтно) разбирает пакет: сверяет номер (`seq` и его дополнение `~seq`), пишет данные в память, считает **CRC16-CCITT** и сверяет с двумя хвостовыми байтами:

```c
} else if (rxsize < pktsize) {
    if (totalbytes + pktsize <= destsz)
        buf[rxsize] = c;                              // ← ЗАПИСЬ БАЙТА В ПАМЯТЬ (dest)
    partial_crc = crc16_ccitt_update(partial_crc, c);
}
...
if (seq[0] != (uint8_t)~seq[1])       { _outbyte(NAK); ... }   // ошибка номера
else if (buf_crc != partial_crc)      { _outbyte(NAK); ... }   // ошибка CRC → переслать пакет
else if (seqnum == seq[0]) {
    ++seqnum;
    buf += pktsize;                                            // следующий пакет пишем дальше
    totalbytes += pktsize;
    _outbyte(ACK);                                            // ← подтверждение пакета
}
```

**3. Квитирование.** На корректный пакет — `ACK` (0x06), на битый (плохой CRC/номер) — `NAK` (0x15), и отправитель шлёт пакет заново (до `MAXRETRANS = 25` попыток). Конец передачи — `EOT` (0x04), приёмник отвечает `ACK` и возвращает число принятых байт.

### Куда именно пишется бинарник

Байты пишутся **напрямую в память по адресу `dest`**, который вы ввели после команды `1`. Практически:

- **DDR** (`0x00000000`) — для больших программ (главная память, до 128 МБ).
- **TCM** (`0xF0000000`) — для маленьких быстрых программ. Например, штатный `sw/hello` слинкован именно под TCM:
  ```
  // sw/hello/common/tcm.ld
  MEMORY { TCM (rwx) : ORIGIN = 0xF0000000, LENGTH = 64K }
  ```
  Значит `hello.bin` грузят по адресу `0xF0000000` и оттуда же запускают.

> Важно: **адрес загрузки должен совпадать с адресом линковки** вашего ELF/bin. XMODEM переносит «сырой» образ (`objcopy -O binary`), без информации об адресах, — поэтому программу кладут ровно туда, под какой её собрал линкер.

## 7. Запуск загруженного бинарника

Команда меню `g` (`cmd_start`) с тем же адресом:

```c
static void cmd_start(void *arg) {
    uart_putc('\n');
    sc1f_uart_tx_flush();          // дождаться, пока UART допередаёт
    rtc_delay_us(20000);           // небольшая пауза
    asm volatile ("jalr %0" :: "r"(arg));   // ← ПРЫЖОК на загруженный код
}
```

`jalr` передаёт управление на введённый адрес — с этого момента исполняется **ваша** программа (её `_start` / `crt`), а загрузчик остаётся «под ней» в OCRAM (программа может вернуться в него, если сделает `ret`, но обычно просто работает).

Полный список команд загрузчика (из таблицы `scbl_commands`):

| Клавиша | Действие |
|---|---|
| `1 @addr` | принять бинарник по XMODEM в память по адресу |
| `g @addr` | запустить (`jalr`) с адреса |
| `d @addr` | дамп памяти |
| `m @addr` | изменить слово в памяти |
| `i` | информация о платформе (ISA, частоты, карта памяти) |

## 8. Практический маршрут «с нуля» (что делает человек)

1. **Собрать битстрим** в Vivado (`source nexys4ddr_scr1.tcl` → Generate Bitstream), затем **вшить загрузчик**: `source ../mem_update.tcl` → получить `..._top_new.bit`.
2. **Прошить** `..._top_new.bit` в плату (или `.mcs` во FLASH).
3. **Собрать программу**: например `cd sw/hello && make` → `hello.bin` (`objcopy -O binary`), слинкованный под `0xF0000000`.
4. **Открыть терминал** на ПК на COM-порту платы (FTDI) с настройками **115200 8N1**. После сброса появится баннер `SCR loader v1.2-...` и меню.
5. **Загрузить**: нажать `1`, ввести адрес `F0000000`, затем в терминале запустить отправку файла по **XMODEM** (`hello.bin`). Дождаться `Xmodem successfully received N bytes`.
6. **Запустить**: нажать `g`, ввести адрес `F0000000`. Программа стартует и печатает свой вывод в тот же терминал (тот же UART).

## 9. Итог

На плате в встроенной SRAM (`0xFFFF0000`, 64 КБ) прошит первичный загрузчик **SCR Loader** — он попадает туда из файла `scbl.mem` через `updatemem` после генерации битстрима. После сброса SCR1 стартует с reset vector `0xFFFFFF00` (это и есть OCRAM с загрузчиком), тот поднимает **UART на 115200 бод, 8N1** (опорный клок 30 МГц, делитель 16) и ждёт команд. По команде `1` загрузчик принимает бинарник по протоколу **XMODEM** (пакеты 128/1024 байта, CRC16-CCITT, ACK/NAK) и **пишет его байт за байтом прямо в память** по указанному адресу (DDR `0x0` или TCM `0xF0000000`). По команде `g` он делает `jalr` на этот адрес — и загруженная программа начинает исполняться, общаясь через тот же UART. Скорость поступления данных ограничена линией UART: ~11 КБ/с.
